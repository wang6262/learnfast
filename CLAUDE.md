# CLAUDE.md 可学习型代码规范（零基础可读 + 进阶成长）
## 一、总体核心规则（最高优先级）
所有本次会话生成、修改的所有代码文件，遵循核心原则：代码可以进阶、可以有深度、可以用于实战学习，但必须保证零基础能看懂、能理解、能复用。
摒弃“只能写简单代码”的限制，适配编程学习、项目实战、源码研读、技术进阶场景，坚守四条铁律：
1. 所有代码必须配分层注释：基础逻辑大白话讲解，进阶语法、核心原理单独注解，兼顾小白读懂、进阶学习者吃透原理。
2. 复杂进阶语法允许使用，但必须拆解讲透：不禁止高阶写法、工程化逻辑、算法逻辑，禁止“无注释、无解释、看不懂的黑魔法代码”。
3. 代码兼顾实用性与学习性：不仅能运行，还能让使用者通过代码学到知识点、语法、工程规范。
4. 逻辑完整闭环：进阶代码必须配套原理说明、适用场景、优缺点、替代写法。

## 二、注释强制标准（学习型专属，分层适配）
注释不再只适配简单代码，分为**基础易懂注释（给零基础）+进阶原理注释（给学习者）**，双向兼顾。

### 1. 文件头部注释（每个代码文件必写）
必须包含：文件功能、核心知识点、适用场景、零基础使用方式、进阶技术点。
```
# ==============================================
# 文件名：xxx.py
# 基础功能：【大白话描述文件核心作用，零基础可懂】
# 核心学习知识点：【本文件涉及的语法/算法/工程知识点，供学习使用】
# 适用场景：【日常使用/项目实战/算法练习/工具开发】
# 使用方法：【零基础分步运行教程】
# 进阶说明：【代码优化空间、技术原理、拓展方向】
# 常用配套函数：【本业务场景高频搭配的其他内置/自定义函数清单+简要作用】
# ==============================================
```

### 2. 函数注释（每个函数必写）
基础信息保证小白看懂，新增学习向注解、**函数调用示例、同场景常用替代函数说明**，适配进阶学习需求。
```
def 函数名(参数):
    """
    【基础功能】大白话讲解函数实现的核心效果
    【学习知识点】本函数用到的核心语法、逻辑、技术思想
    参数：
        变量名：【基础释义：数据类型、传入要求】【进阶释义：参数设计原理、适配场景】
    返回值：【基础释义：返回数据含义】【进阶释义：返回设计逻辑、复用价值】
    调用示例：
        示例1：极简基础调用方式（零基础可直接复制运行）
        示例2：进阶参数调用方式（实战常用写法）
    同场景常用替代函数：
        函数名1：作用、和当前函数的差异、适用场景
        函数名2：作用、和当前函数的差异、适用场景
    注意事项：【运行避坑、进阶使用限制】
    """
```

### 3. 行级/块级注释（分层标注）
所有代码块必须分层注释，区分基础逻辑与进阶逻辑，兼顾阅读与学习：
- 基础代码行（变量定义、基础判断、循环、文件读写）：大白话注释，说明「做了什么、作用是什么」，保证零基础读懂。
- 进阶代码块（高阶语法、算法逻辑、异常捕获、路径安全校验、性能优化）：额外增加原理注释，说明「为什么这么写、对比普通写法的优势、适用场景」。
- 核心难点代码（复杂逻辑、链式调用、算法公式、工程防护逻辑）：单独拆分讲解，补充通俗原理解析、简化替代写法、拓展用法。

核心注释原则：简单代码通俗讲，进阶代码拆解讲，难点代码透彻讲，既能跑通，又能学会。

## 三、代码写法规范（支持进阶学习，不限制简单代码）
### 1. 允许使用的进阶写法（专属学习优化）
完全放开高阶语法限制，用于学习实战，只需配套完整注释讲解：
- 列表/字典推导式、嵌套推导式（需标注基础替代循环写法）
- 三元表达式、链式调用、高阶函数（map/filter/lambda）
- 面向对象、类继承、封装、装饰器、上下文管理器
- 异常嵌套捕获、路径安全校验、正则匹配、批量处理逻辑
- 算法逻辑、数据结构、模型训练、批量工程化处理代码

### 2. 禁止的写法（仅杜绝晦涩无学习性代码）
仅禁止无意义炫技、无法学习、无法阅读的代码：
- 无任何注释的极简压缩黑魔法代码、无意义嵌套简写
- 变量、函数命名晦涩无释义，无任何说明的工程代码
- 逻辑跳跃、无闭环、无容错、不适合新手学习的残缺进阶代码

### 3. 代码结构规范（适配学习）
- 所有代码分步拆分、功能模块化，单一功能独立代码块，方便逐段学习、逐段调试。
- 固定配置、参数、常量统一放在代码顶部，方便修改、对照学习。
- 进阶逻辑后可选择性附带「简易替代写法」，方便新手过渡学习。
- 复杂代码必须附带逻辑流程图/步骤拆解（文字版），梳理核心思路。
- 同功能的多个高频函数集中整理使用说明，放在文件开头或函数文档字符串内，方便横向对比学习。

### 4. 变量与函数命名规范
- 优先使用行业通用规范命名，贴合实战项目（适配学习行业标准）。
- 自定义命名见名知意，所有缩写、专业术语必须配套注释解释。
- 区分日常变量、工程变量、算法变量，贴合不同学习场景。

## 四、学习型交付规范（每次生成代码必带）
所有代码交付，不再只满足“能用”，必须满足“能学、能懂、能进阶”，固定交付结构：
1. 功能概述：大白话讲解代码作用、运行效果。
2. 核心学习知识点：罗列本次代码可学到的语法、逻辑、工程技巧、算法思想。
3. 环境依赖：完整的库安装命令、运行环境要求。
4. 完整分层注释源码：基础注释+进阶原理注释双配套，每个函数附带调用示例+同类常用函数对比。
5. 分步运行教程：零基础可直接照做运行。
6. 难点原理拆解：针对进阶代码、难点逻辑，单独通俗讲解。
7. 常用配套函数汇总：本场景下高频内置函数、第三方函数、自定义工具函数清单，附带作用、优缺点、使用场景。
8. 常见报错与优化方案：报错解决+代码拓展优化思路。
9. 拓展学习方向：给出代码升级、延伸学习的思路，助力进阶。

## 五、日志强制规则（不变延续）
每次修改/新增代码文件，必须在 log.md 末尾追加标准修改日志，使用固定模板：包含日期、文件名、修改目的、参数改动表格、数据依据。
禁止覆盖、删除旧日志，只允许尾部追加，保证项目迭代可追溯。

## 六、标准示范（学习型代码模板，新增调用示例+同类常用函数）
```python
# ==============================================
# 文件名：safe_file_read.py
# 基础功能：安全读取本地文本文件，自动兼容编码错误、路径越界问题
# 核心学习知识点：路径标准化、异常分层捕获、路径安全校验、编码兼容处理
# 适用场景：Python文件读写实战、本地数据处理、工具脚本开发
# 使用方法：直接运行函数，传入本地文件路径即可读取文本
# 进阶说明：代码包含工程级安全防护逻辑，可拓展批量读取、编码自动识别功能
# 常用配套函数：
# 1. Path.write_text()：将字符串写入文本文件，和本函数成对使用
# 2. Path.exists()：判断文件是否存在，读取前做前置校验
# 3. Path.is_file()：判断路径是否为文件，避免误传入文件夹
# ==============================================
from pathlib import Path

def safe_read_text(file_path):
    """
    【基础功能】安全读取任意本地文本文件，自动处理报错，不会程序崩溃
    【学习知识点】
        1. Path路径标准化：resolve()消除路径歧义、防御路径穿越
        2. 分层异常捕获：精准区分编码错误、系统读取错误
        3. 工程容错思想：实战项目必须做异常兜底，提升代码健壮性
    参数：
        file_path: 字符串，支持相对路径/绝对路径，为需要读取的文件地址
    返回值：
        成功：返回完整文件文本字符串
        失败：返回精准中文错误提示，方便排查问题
    调用示例：
        # 示例1：基础调用（零基础）
        content = safe_read_text("test.txt")
        print(content)

        # 示例2：进阶绝对路径调用
        res = safe_read_text(r"C:\Users\524\Desktop\demo.md")
    同场景常用替代函数：
        1. Path.read_bytes()：以二进制方式读取文件，适合图片、PDF等非文本文件
        2. ast.literal_eval()：配合读取字符串格式的字典/列表，安全还原结构化数据
        3. json.load()：专门读取json格式文本，标准化解析JSON数据
    注意事项：
        1. 仅支持文本格式文件读取，二进制文件会触发编码异常
        2. 文件路径不能包含中文特殊字符以外的非法符号，Windows路径建议使用原始字符串r""
    """
    # 进阶逻辑：标准化路径，解析绝对路径、消除../相对路径歧义
    # 学习点：解决不同系统、不同写法的路径报错问题，防御路径穿越攻击
    p = Path(file_path).resolve()

    # 获取当前程序运行根目录，作为路径安全校验的基准
    root_dir = Path.cwd().resolve()

    # 进阶安全校验：限制文件只能在项目根目录内，防止读取系统敏感文件
    # 学习点：工程实战必备安全逻辑，杜绝路径遍历漏洞
    try:
        p.relative_to(root_dir)
    except ValueError:
        return f"安全错误：仅允许读取项目目录内文件，禁止读取外部路径 {file_path}"

    # 分层异常捕获读取文件
    try:
        # 通用UTF-8编码读取，适配绝大多数文本文件
        content = p.read_text(encoding="utf-8")
        return content
    # 精准捕获：非文本文件、编码不匹配导致的解码失败
    except UnicodeDecodeError:
        return "读取失败：该文件不是UTF-8编码文本文件，或为二进制文件（PDF/图片/压缩包等）"
    # 兜底捕获：文件不存在、权限不足、文件夹路径等所有未知错误
    except Exception as e:
        return f"读取异常：{str(e)}"
```

## 七、最终兜底规则
1. 不限制代码简单/复杂、基础/进阶，允许所有实战级、学习级代码；
2. 所有进阶、复杂代码，必须注释讲透、逻辑拆解、原理说明，附带**调用示例+同类常用函数对比**，保证零基础能看懂、学习者能横向拓展学习；
3. 所有生成代码优先满足「可运行、可学习、可拓展、可对比」四大核心需求。
